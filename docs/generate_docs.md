# Інструкція з генерації документації

Цей проект використовує Sphinx для створення та підтримки документації. Документація містить опис архітектури, API та інструкції з використання.

## Передумови

Перед тим як почати генерувати документацію, переконайтеся, що встановлені необхідні бібліотеки (зазвичай вони є у `requirements.txt`):

```bash
# Активація віртуального середовища
# Наприклад:
# source .venv/bin/activate  # (Linux/macOS)
# .venv\Scripts\activate     # (Windows)

pip install sphinx
```

## Використання Makefile (Автоматизований спосіб)

Швидкий та рекомендований спосіб збірки документації — використання команди `make` в кореневій директорії проекту.

```bash
make docs
```

Ця команда автоматично:
1. Перейде до директорії `docs`
2. Викличе компілятор Sphinx для побудови HTML сторінок.
3. Розмістить результат у папку `docs/build/html/uk`.

Після успішного виконання команди, згенерована документація буде доступна локально.
Відкрийте цей файл у веб-браузері:
`docs/build/html/uk/index.html`

## Ручна генерація

Якщо у вас немає встановленого інструменту `make` (наприклад, ви на Windows без встановленого make), ви можете виконати команди власноруч:

```bash
# Перейдіть в папку docs
cd docs

# Запустіть генерацію HTML
python -m sphinx -b html source build/html/uk
```

## Автоматична публікація

Для цього проекту також налаштовано автоматичний пайплайн CI/CD за допомогою GitHub Actions. При кожному пуші (або злитті Pull Request) у гілку `main` автоматично відбувається генерація найсвіжішої документації та її публікація на GitHub Pages. Це гарантує, що онлайн-версія завжди актуальна.
