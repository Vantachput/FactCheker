Module for handling text messages and fact-check requests.

Business Logic (Verification & Fallback): When a user sends a news item (as plain text or a forwarded message), this module: 1. Parses the source (channel/group forward or direct). 2. Checks user state and limits. 3. For the base method, first generates a smart search query. If that query

yields no Google results (e.g. the fake is very abstract), the Fallback Algorithm kicks in: the raw news text (first words) is sent directly to Google to find any related debunking.

  1. Splits large responses (over 4000 characters) into parts via send_smart_reply.

async handlers.message_handlers.handle_message(update, context, user_states: dict)[source]

Main content handler for user-submitted news.

Checks if the user is in the WAITING state (i.e. pressed ‘Classify News’). If so, extracts the text or media caption, identifies the forward source, routes the request through the selected AI method (Together, OpenAI, Serper, Perplexity), and returns the verdict.

Parameters:

  • update – Update request from the client (contains the message).

  • context – Bot context from telegram.ext.

  • user_states (dict) – User state dictionary.

async handlers.message_handlers.send_smart_reply(update, text: str, status_msg=None)[source]

Sends large messages in parts.

Since Telegram limits a single message to 4096 characters, this function splits the text into chunks, ensuring that the Link Preview only appears on the first message (to avoid spam).

Parameters:

  • update – Update request.

  • text (str) – Large text to send.

  • status_msg (optional) – Status message (“Analyzing…”) to be replaced with the result. If None, sent as a reply.
