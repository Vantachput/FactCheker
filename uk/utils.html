Utilities

Helper utilities for text processing and time handling.

This module contains pure functions used for formatting responses, splitting text for Telegram, escaping special characters, and working with Ukrainian local time.

Tests (doctest):

You can run the tests for this module with: python -m doctest utils/helpers.py -v

utils.helpers.escape_markdown(text: str) → str[source]

Escapes special Markdown characters for safe sending in Telegram.

Telegram bots may raise a ParseError if a message contains unmatched or incorrectly formatted Markdown characters.

Parameters:

text (str) – Original text.

Returns:

Text with escaped _, *, [ characters.

Return type:

str

Examples

>>> escape_markdown("Текст з _курсивом_ та *жирним* шрифтом [Лінк]")
'Текст з \\_курсивом\\_ та \\*жирним\\* шрифтом \\[Лінк]'
utils.helpers.get_progress_bar(text: str) → str[source]

Generates a visual AI confidence bar (up to 10 blocks).

The function searches for a percentage mention in the text (e.g. “85%”) and converts it into a string of emoji squares.

Parameters:

text (str) – Input text from AI containing the confidence percentage.

Returns:

Formatted string with the bar, or an empty string if no percentage is found.

Return type:

str

Examples

>>> get_progress_bar("Впевненість: 87%")
'\n\n📊 Шкала впевненості ШІ:\n🟩🟩🟩🟩🟩🟩🟩🟩⬜⬜ 87%'
>>> get_progress_bar("Без відсотків")
''

utils.helpers.split_text(text: str, max_length: int = 4000) → list[str][source]

Splits text into chunks of the specified length.

Telegram limits a single message to 4096 characters. This function reliably splits large text into parts.

Parameters:

  • text (str) – Input text to split.

  • max_length (int, optional) – Maximum length of a single chunk. Default is 4000.

Returns:

List of text chunks.

Return type:

list[str]

Examples

>>> split_text("1234567890", max_length=3)
['123', '456', '789', '0']
>>> split_text("", max_length=5)
[]

Bot cost analytics and logging system.

Business Logic (Cost Tracking): This module records all AI usage statistics. Each request is converted into token counts, then translated into USD using provider pricing (OpenAI, Together). Data is stored in two formats: 1. bot_usage.log — plain text log for quick admin review. 2. usage_analytics.jsonl — structured JSON Lines file for deep

machine analytics or chart generation.

async utils.logger.log_ai_usage(method: str, model_name: str, usage_data: object, user_id: str | int = 'Unknown')[source]

Universal async logger for collecting token usage and cost statistics.

The function adapts to different usage_data formats (plain dict or Pydantic model from OpenAI/Together). It identifies the model by name and automatically applies the current per-million-token pricing to calculate the final cost.

Parameters:

  • method (str) – Call method (e.g. BASE, PERPLEXITY).

  • model_name (str) – Exact model name (e.g. gpt-4o-mini).

  • usage_data (object | dict) – Object containing the number of tokens used.

  • user_id (str | int, optional) – Telegram user ID.
