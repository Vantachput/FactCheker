База Даних (Database)

Модуль для роботи з локальною базою даних SQLite.

Цей модуль відповідає за ініціалізацію та підтримку постійного (persistent) з’єднання з базою даних bot_data.db. Основне призначення БД — це ведення обліку використання користувачами платних моделей штучного інтелекту, встановлення денних лімітів та їх обнулення.

async database.db_manager.check_and_increment_limit(user_id, model_name, admin_id)[source]

Перевіряє ліміти використання моделі та інкрементує лічильник.

Якщо ліміт вичерпано, або виникла помилка підключення — повертає False. Для безкоштовних моделей (наприклад «sonar») або для адміністратора перевірка завжди успішна і лічильник не збільшується. Ліміти автоматично оновлюються (скидаються) на початку нової доби.

Параметри:

  • user_id (int) – ID користувача Telegram.

  • model_name (str) – Назва моделі (наприклад sonar-reasoning-pro).

  • admin_id (int | str) – ID адміністратора бота (перевіряється на співпадіння з user_id).

Повертає:

Кортеж з двох значень:

  • bool: True, якщо запит дозволено, False — якщо ліміт вичерпано.

  • int | None: Залишок або денний ліміт. Для успішних або безлімітних запитів зазвичай None.

Тип повернення:

tuple[bool, int | None]

async database.db_manager.close_db()[source]

Коректно закриває активне з’єднання з базою даних.

Ця функція повинна викликатися при зупинці роботи бота.

async database.db_manager.init_db()[source]

Ініціалізує та відкриває постійне з’єднання з базою даних.

Створює таблицю usage, якщо вона ще не існує. Таблиця зберігає user_id, model_name, count (кількість запитів) та last_reset (дату останнього обнулення).

Зверніть увагу, що ми не використовуємо контекстний менеджер async with, оскільки нам потрібно зберегти з’єднання відкритим у глобальній змінній _db_conn для швидкого доступу під час обробки повідомлень.
